# Задача проекта

Хочу в опенкод писать текстом и голосом с телефона например телеграм или другое приложение.

Опенкод работает на сервере и получает запросы мои и присылает ответы мне

Схема выглядит так:

- Вы пишете сообщение в Telegram с телефона.
- Бот на сервере получает его через Telegram API.
- Если это голос, бот сначала делает speech-to-text.
- Затем текст отправляется в OpenCode.
- OpenCode выполняет задачу и возвращает ответ.
- Бот присылает результат вам обратно в Telegram.

# Безопасность

- логины, токены, пароли, .env коммитить запрещено
- если очень надо сохранить где-то креды, добавь этот файл в .gitignore
- или храни вне гит проекта, тут %USERHOME%/.rem-opencode/config.json

# Технологии

- телеграм бот на питоне
- фильтр по id пользователя
- все компоненты запускаются в docker compose
- shell run.sh и stop.sh  котрый запускается и на линукс и на виндрус в среде mingw git bash
- контейнер с опенкодом должен содержать: node js 21, jdk  java 21 и gradle 8.7, python3.9, opencode cli установленный через npm, и плагин context7, обертку http api wrapper так как образа готового нет
- конфиг Opencode.json внутри в образе есть, но может быть переопределен через вольюм
- кеш гредла пусть во внешний вольюм наполняется, чтобы при каждом запуске контейнера не ждать

# Окружение

- В этой папке создай git проекта
- токен для работы с телеграмом должен лижать в %USERHOME%/.rem-opencode/config.json
- рабочая директория монтируется в контейнер опенкода через вольюм
- создай файл mount/AGENTS.md который будет монтироваться в контейнер с опенкодом как глобальный агент
- обязательно в mount/AGENTS.md укажи агенту информацию, что он в контейнере работает, и должен выбирать решения исходя из возможностей
- создай папку mount/SKILLS/<имя скила> она должна монтироваться в образ и создай четыре скила: сеньер разработчик на джава, тестировщик кукумбер джава, сеньер разработчик на питоне, тестировщик кукумбер питон
- дособери официальный образ опенкода и добавь в него jdk  java 21 и gradle 8.7 чтобы при помощи его запускались автотесты и компилировался код, чтобы опенкод в контейнере проверять мог себя
- если будет требоваться частая инсталяция чего-то при выполнении задач опенкодом, пусть
